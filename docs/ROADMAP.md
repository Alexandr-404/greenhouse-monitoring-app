# Пожелания и рекомендации по развитию

## 1. Улучшения API (важно для UI/производительности)

1. **Агрегация статусов**

- Сейчас "светофор" строится по всем теплицам на клиенте.
- Рекомендация: добавить endpoint типа:
  - `GET /states/summary?region_id=<id>` или `GET /greenhouses/summary`
  - Ответ: counts по статусам + last_updated

2. **Пагинация и фильтрация на бекенде**

- Для больших списков теплиц/регионов UI будет тяжёлым.
- Рекомендация:
  - `GET /greenhouses?region_id=&q=&page=&page_size=`

3. **Единый endpoint “карточка теплицы”**

- Чтобы не дергать много запросов, добавить:
  - `GET /greenhouses/:id/details` (метаданные + last_state + last_update + summary)

## 2. Улучшения UI/UX

1. Разные режимы отображения графиков (Line/Area, сравнение показателей)
2. Выбор произвольного диапазона дат (date range picker)
3. Экспорт данных (CSV/Excel) по измерениям и состояниям
4. Настраиваемые пороги/алерты (уведомления)
5. Сохранение пользовательских пресетов (показатель+период+фильтры)
6. Клавиатурная навигация

## 3. Безопасность и доступ

1. Реальная авторизация (OIDC/SSO) и выдача ролей с бекенда
2. Аудит-лог изменений:

- кто изменил измерение, когда, почему (обязательное поле “причина”)
- кто оставил комментарий к состоянию

## 4. Наблюдаемость и качество

1. Sentry/логирование ошибок фронта
2. Метрики производительности (Web Vitals)
3. Тесты критичных сценариев (выбор теплицы, график, редактирование)
